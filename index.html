Frequently Asked Questions

1. Write the differences between var, let and const.

In Javascript, var, let and const are used for declaring variables. But they have dissimilarities in terms of scope and mutability.
Var: var is bascially function scoped or global if declared outside and can be updated and re-declared. We can access var declared variables from outside of an function.
let: let is basically block scoped, means that it can only be accessed within the curly braces {}. It can be reassigned but can't be re-declared within the same scope.
const: const are block scoped also but can't be reassigned and re-declared. If they are accessed before initialization, referenceError will occur.

2. Write the differences between map(), forEach(), and filter().

In Javascript, map(), filter(), and forEach() are array methods for manipulating array. However, their behaviour are quite different from one another while applied in a array.
map(): map() method always returns a new array after modifying the elements of the original array. map() works on every elements of the original array and after manipulating the elements, it returns them in a new array. This method is used for modifying all elements of an array.
filter(): filter() method also return a new array but only with the elements that match the requirement. This method is mainly used for filtering data of an array. Based on the condition given, it filters the elements of an array and returns them in a new array.
forEach(): forEach() method doesn't return anything. It receives some condition as arguments and applied them to each elements of that array. Thus, it returns undefined but only applies the given conditions to each element of the original array.

3. Explain arrow functions and how they are different from regular functions.

Arrow Function: Arrow function in Javascript is a shorter way of writing function which is introduced in ECMAScript 6. It allows us to write fucntion code in a more concise and readable way. Unlike regular functions, arrow functions don’t have their own this, but instead, inherit it from the surrounding context.
Sample arrow function-- let functionName = (parameter) => (function body);
Arrow functions and regular functions in JavaScript serve similar purposes, but they differ in terms of syntax, behavior, use cases and many others. There are certain interesting differences between them as discussed below —
No arguments: Arrow functions do not have their own arguments object like the regular functions.
Cannot be used as constructors: Arrow functions cannot be used with the new keyword.
Return Keyword: Unlike regular functions, return doesn't need to be used in an arrow function if function body consists of one line.
These are some of the common differences between regular function and arrow function although there are some more differences between them.

4. How JavaScript Promises work?

In JavaScript, a promise is an asynchronous action that may complete at some point and produce a value. Promise simplify the use of asynchronous functions.
Moreover, a Promise is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. It can be in one of three states:
1. Pending: The initial state, neither fulfilled nor rejected.
2. Fulfilled: The operation completed successfully, and the promise has a resulting value.
3. Rejected: The operation failed, and the promise has a reason for the failure (typically an error).
Example: While fetching an api data, a promise is occured and we can use .then for fullfilled promise and .catch for rejected promise. code snippet -
fetch('some api url')
.then(res => res.json()) //a promise that convert the json data
.then(data => console.log(data)) //used to show the data in the console
.catch((error) => console.log(error)) //if promise is failed it will tells us the error.
 So, in the above example, after fetching the url, a promise is occured in the second line. Thus, in the third and fourth line by chaining them with the .then and .catch. Like this way, promises are made to perform a series of asynchronous operations one after the other.
With the introduction of async and await, working with Promises became even more straightforward, but understanding how Promises work at their core is essential for effective JavaScript programming.

5. How closures work in JavaScript?

In JavaScript, closure is a function that remembers its outer variables and can access them.
function multiplier(factor) {
return number => number * factor;
}
let twice = multiplier(2);
console.log(twice(5));
//output is 10
In the above example, multiplier is called and creates an environment in which its factor parameter is bound to 2. The function value it returns, which is stored in twice, remebers this environment. So, when that is called, it multiplies its argument by 2.
Thus, by remembering its outer variables, function can access them and update them.
This is how closure is being used and defined in JavaScript.
